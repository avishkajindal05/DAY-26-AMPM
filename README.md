## 🎯 Problem Statement

Pharmalink aims to optimize its operations by understanding:

* Customer ordering behavior
* Impact of recency (`days_since_last_order`)
* Variations across different **city tiers**
* Retention and engagement patterns

The goal is to build insights that help:

* Improve customer retention
* Personalize engagement strategies
* Optimize operational efficiency

---

## 🔍 Key Concepts Used

### 1. Interaction Features

* Example:

  ```
  days_since_last_order × city_tier
  ```
* **Purpose:** Capture behavioral differences across regions within a single model

---

### 2. Customer Segmentation

* Based on recency and activity
* Helps in targeted marketing and retention strategies

---

### 3. Behavioral Analysis

* Identifying how frequently users reorder
* Understanding drop-off patterns

---

### 4. Scalability Consideration

* Instead of separate models per city tier, a **single model with interaction features** is used
* Reduces complexity while maintaining performance

---

## 📊 Approach

* Problem Understanding
* Feature Engineering (interaction-based)
* Business-Oriented Reasoning
* Structured Case Study Explanation

---

## 🚀 Key Insights

* Customer behavior varies significantly across **city tiers**
* Recency is a strong indicator of churn risk
* Interaction features improve model flexibility without increasing system complexity
* Targeted interventions can improve retention
